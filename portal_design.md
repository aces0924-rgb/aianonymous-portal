# イベントポータル汎用化 ＆ 管理者設定ページ設計書 (6/6以降用)

本ドキュメントは、6月6日の結果発表以降に実施する、イベントポータルサイト化（マルチテナント化）およびイベント管理者向け設定ページの設計案をまとめた資料です。

---

## 1. ポータル化のコンセプト

- **目的**: 1つのシステム（Next.js/Prisma）で、複数の異なる音楽イベント特設サイトをノンプログラミングで立ち上げ・運用できる「イベント作成プラットフォーム（SaaS型）」へのアップグレード。
- **アプローチ**:
  - 各イベントをデータベースの `Event` レコードとして分離する。
  - 各ページのレイアウトや機能のON/OFF、表示文言を、データベースに保存された設定値から動的に生成する。

---

## 2. データベース設計 (マルチテナント対応)

すべてのテーブルに親となる `Event` テーブルとの紐付け（`eventId`）を追加し、複数イベントのデータが混ざらないように設計します。

```prisma
// 新設する最上位の親テーブル
model Event {
  id          String   @id @default(uuid())
  slug        String   @unique // URLのパスに使用 (例: "anonymous-fes", "musume-fes")
  title       String   // イベント名
  description String?  @db.Text
  themeConfig String   // 配色やグラデーションなどのCSSスタイル用JSON
  featureFlags String  // 各種機能のON/OFFスイッチ用JSON
  labelConfig String   // 各項目のカスタマイズラベル用JSON
  
  // 各子テーブルとのリレーション
  tracks      Track[]
  news        News[]
  schedules   Schedule[]
  faqs        Faq[]
  
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

// 既存モデルへの eventId 紐付け例（Track）
model Track {
  id        Int      @id @default(autoincrement())
  eventId   String
  event     Event    @relation(fields: [eventId], references: [id], onDelete: Cascade)
  title     String
  songUrl   String
  // ... (その他の既存フィールドは維持)
}
```

## 3. イベント管理者用設定ページ (管理画面 `/admin/events/[id]/settings`)

非エンジニアのイベント管理者が、Web画面から直感的に特設サイトをカスタマイズ・設定できるUI構成案です。

### 設定フォーム項目一覧

| タブカテゴリ | 設定項目名 | 入力タイプ | 説明・挙動 |
| :--- | :--- | :--- | :--- |
| **基本情報** | イベント名 | テキスト入力 | サイトのタイトルタグやメイン看板に反映。 |
| | URLスラッグ | 英数字入力 | `/[slug]` でアクセスできるようになります。 |
| | ロゴ画像 | ファイルアップロード | サイトヘッダー・フッターに表示する画像。 |
| | メイン背景画像 | ファイルアップロード | ヒーローエリア（看板部分）の背景画像。 |
| **デザイン** | メインカラー | カラーピッカー | サイト全体のキーカラー（現在はシアン `#00f0ff`）を自由に変更。 |
| | グラデーション | カラーピッカー×2 | ボタンや枠線に使うサイバーパンク風グラデーション。 |
| **機能ON/OFF** | サムネイル投稿機能 | トグルスイッチ | `ON` にすると、楽曲詳細に「サムネイル投稿する」ボタンが出現。 |
| | 推しリスト（お気に入り） | トグルスイッチ | ユーザーが「推しリスト」を作成して共有できる機能。 |
| | YouTube再生リスト | トグルスイッチ | ヒーローエリアに「YouTube再生リスト」へのリンクボタンを配置。 |
| | ランダム再生 | トグルスイッチ | ヒーローエリアに「ランダムで曲を聴く」ボタンを配置。 |
| **文言変更** | 歌詞 / クレジットラベル | テキスト入力 | デフォルトの「LYRICS」を変更（例: 「クレジット」「解説」など）。 |
| | 考察タブ1 ラベル | テキスト入力 | デフォルトの「歌詞考察」を変更（例: **「アーティスト概要」**など）。 |
| | 考察タブ1 注意書き | テキスト入力 | タブの横に小さく表示されるAI考察などの免責・注記文。 |
| | 免責事項（Disclaimer） | テキストエリア | 楽曲詳細に表示する「特定防止」等のポリシー注意文。 |

## 4. ルーティング（URL設計）

イベント管理者が登録した `slug` をベースにした Next.js の動的ルーティング（Dynamic Routes）を採用します。

- **`/[eventSlug]`**: イベントごとのトップページ
- **`/[eventSlug]/tracks`**: 該当イベントの楽曲一覧ページ
- **`/[eventSlug]/tracks/[id]`**: 該当イベントの楽曲詳細ページ
- **`/[eventSlug]/submit-thumbnail`**: サムネイル投稿フォーム
- **`/admin`**: 全イベントを統括するシステム管理者ダッシュボード
- **`/admin/events/[id]/settings`**: 個別イベントの主催者用設定ページ
