# AIアノニマスフェス ポータルサイト システム仕様・運用マニュアル

本ドキュメントは、現在運用しているイベントポータルサイト（aianonymous-portal）のシステム仕様および、開発・更新の運用フローについてまとめたものです。

---

## 1. システム概要・技術スタック

本ポータルサイトは、最新のWebフロントエンド技術を用いて構築された、高パフォーマンスかつカスタマイズ性の高いWebアプリケーションです。

- **フレームワーク**: Next.js (App Routerベース)
- **開発言語**: TypeScript / JavaScript
- **スタイリング**: Tailwind CSS (ダークテーマ・ネオンエフェクト対応)
- **データベース ORM**: Prisma
- **データベース**: PostgreSQL (Supabase等を想定)
- **ホスティング・デプロイ**: Vercel

---

## 2. 主要機能・仕様

### 2.1 マルチイベント対応（動的ルーティング）
- `/[eventSlug]` というURL構造により、1つのシステムで複数のイベント（第1回、第2回、イラスト部門、他イベントなど）を並行して運用可能です。
- イベントごとにデータベース上で「テーマカラー」「ロゴ」「背景画像」などを設定（ThemeConfig）でき、個別のデザインを適用できます。

### 2.2 応募システム
- **フォーム設定**: 音楽用（匿名/記名）、イラスト用など、イベント設定（FeatureFlags）によってフォームの入力項目を動的に切り替え可能です。
- **ファイルアップロード**: 音源（mp3/wav）や字幕（srt）などを直接アップロードし、Google Drive等へ保存する機能を持っています。

### 2.3 トラック（作品）視聴・閲覧システム
- **一覧・検索機能**: 「推しリスト」「参加作品一覧」や、ナビゲーションバーの「JUMP（番号・名前検索）」により、大量の作品からスムーズに検索できます。
- **プレイヤー機能**: YouTubeリンク、Suno短縮リンク、直接アップロードされた音声ファイルなど、様々な音源形式を再生できるカスタムオーディオプレイヤーを搭載しています。
- **お気に入り（推し）機能**: 気になった作品をローカルに保存し、一覧化できる推し機能（FavoritesContext）を備えています。

### 2.4 管理機能 (Admin)
- `/admin` ルートから、イベントの作成、スケジュールやFAQの更新、応募されたトラック情報の管理、サイトの機能ON/OFF（投票モードの切り替えなど）を一括で管理できます。

## 3. 運用・開発フロー（更新手順）

本サイトは、**「ローカルでの確認」→「GitHubへの保存」→「Vercelへの自動反映」**というモダンなCI/CD（継続的インテグレーション/デプロイ）フローを採用しています。

### STEP 1: ローカル環境での開発・デザイン確認
デザインの変更や機能の追加は、まずローカル（手元のPC）で開発サーバーを立ち上げて行います。

1. ターミナルで `npm run dev` を実行し、ローカルサーバーを起動します。
2. ブラウザで `http://localhost:3000` (または 3002 など) にアクセスします。
3. コード（`.tsx` や `.css`）を書き換えると、即座にブラウザの画面が自動更新（ホットリロード）されます。ここでデザインや動作の確認・デバッグを行います。

### STEP 2: GitHubへのプッシュ（バージョン管理）
ローカルでの動作確認が完了したら、変更したコードを「GitHub」というクラウド上のコード保管庫に送信（プッシュ）します。これにより、変更の履歴が安全に保存されます。

1. **変更の追加**: `git add .` (変更したファイルをすべてステージング)
2. **コミット**: `git commit -m "変更内容のメモ（例：ヘッダーの色を変更）"`
3. **プッシュ**: `git push origin main` (GitHubのメインブランチへ送信)

### STEP 3: Vercelでの自動デプロイ（本番環境への適用）
本番環境のサーバーである「Vercel」は、GitHubと連携しています。

1. GitHubに新しいコードがプッシュされると、Vercelがそれを自動的に検知します。
2. Vercelのサーバー内で自動的にビルド（最適化処理）が開始されます。
3. **数分以内**に処理が完了し、実際のURL（`https://aianonymous-portal.vercel.app/` 等）に最新の状態が自動的に反映されます。

> **💡 運用上のポイント**  
> Vercelはアクセス負荷に非常に強いため、フェス開催中や結果発表時など、急激にアクセスが集中してもサーバーが落ちにくいという大きなメリットがあります。

---

## 4. データベースの更新について
もしデータベースの構造（Prisma Schema）を変更した場合は、以下の手順が必要になります。
1. ローカルで `npx prisma db push` を実行し、データベースに変更を反映
2. `npx prisma generate` を実行し、コード側で最新のDB構造を認識させる
3. その後、STEP 2と同様にGitHubへプッシュすることで、Vercel上でも最新のDB構造を用いたビルドが行われます。
