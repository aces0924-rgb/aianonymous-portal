## 1. 最近の重要修正事項

### 🎵 グローバルオーディオプレイヤーの不具合修正
- **事象**: 楽曲一覧画面で再生ボタンを押すと、プレイヤーが一瞬表示されてすぐに消えてしまう。
- **原因**: 
    1. `PlayerContext.tsx` の関数が再レンダリングのたびに作り直されていた（参照不一致）。
    2. `GlobalPlayer.tsx` の `useEffect` がその参照不一致を「ページ遷移」と誤認してリセットを実行していた。
- **解決策**:
    - `PlayerContext.tsx` で `playTrack` と `closePlayer` を `useCallback` でメモ化。
    - `GlobalPlayer.tsx` に `useRef` を導入し、実際に `pathname` が変更された時のみリセットがかかるようガードを追加。

### 🖼️ サムネイル投稿機能の一般公開対応
- **変更点**:
    - 楽曲詳細画面の「サムネイルを投稿する」ボタンの `preview=honban` 制限を解除。
    - 投稿フォーム（`ThumbnailSubmissionForm`）内の戻り先リンクを動的に変更（プレビューモードならプレビュー用一覧、通常なら通常一覧へ）。
    - 投稿時のファイルサイズ上限を **4MB** に設定（UX向上のため）。

### 📊 データベース・バックエンド
- **スキーマ拡張**: 
    - `TrackThumbnail` モデルに `entryNo`（エントリー番号）と `trackTitle`（楽曲名）を追加。管理者がDBを直接見てどの曲の投稿か判別可能に。
- **同期処理**:
    - `Track`（選考用）から `TrackHonban`（本番用）へのデータ移行ツール（`/admin/repair-db`）を作成・実行済み。
    - **注意**: スキーマ変更後は `npx prisma db push` および `npx prisma generate` が必須です。

## 2. UI/UX 改善事項

- **歌詞表示**: 最大高さを `460px` から `600px` に拡大し、スクロールなしで読める量を増加。
- **ナビゲーション**: 
    - 楽曲詳細画面の左上ボタンを「◀ フェス会場へ 戻る」から「◀ 参加作品一覧へ」に変更。
    - 遷移先もトップページ（`/`）ではなく、一覧ページ（`/tracks`）へ直接戻るように修正。
- **絞り込み**: 「ファンアート募集中」の絞り込みとバッジを、一般ユーザーにも開放。

## 3. モバイル端末対応 (重要)

- **予約枠の解除**: モバイルユーザーがブラウザを閉じたり戻ったりした際に、確実に予約を解除するための仕組みを実装。
    - `pagehide` イベントと `fetch(..., { keepalive: true })` を組み合わせ、接続が切れる寸前にサーバーへ信号を送るようにしています。
